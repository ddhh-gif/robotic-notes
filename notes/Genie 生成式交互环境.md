**论文** Genie: Generative Interactive Environments，Google DeepMind，arXiv:2402.15391v1（2024-02-26）。11B 参数，对外定位"foundation world model"。本地文件 `genie.pdf`。

**一句话本质** 它只看画面、不看按键记录，自己猜出"两帧之间玩家做了什么动作"。猜出 8 个就够（等于一个 8 键手柄），学会后给它任意一张图就能"按按钮"玩进去。整篇论文真正的魔法只有一处：在**没有动作标签**的情况下，把"动作"这个概念从像素里逼出来。

**核心问题：没有动作标签怎么学控制** 一般世界模型要"视频 + 动作"成对数据：每帧配一个"玩家按了上/下/跳"。这种数据互联网上几乎没有，标注又贵。Genie 的赌注：把"动作"当成一个**待学的隐变量**。
做法（隐动作模型 LAM）：编码器吃进过去所有帧 $x_{1:t}$ 加上**下一帧** $x_{t+1}$，吐出连续隐动作 $\tilde a_t$；解码器只拿过去帧 + 这个隐动作去重建 $\hat x_{t+1}$。关键约束：解码器**看不到真正的未来帧**，所以隐动作 $\tilde a_t$ 必须把"从现在到未来最关键的变化"压进去——否则重建不出来。这就逼着它学到"动作 = 帧间的因果差异"。
再用 VQ-VAE 把隐动作量化到一个**只有 8 个码**的码本（$|A|=8$）。为什么要这么小？因为人要能玩——8 个动作就是 8 个按钮。瓶颈越窄，越逼模型只留"真正能操控的东西"，扔掉无关画面变化。
训练完，整个 LAM 除了那个 8 码码本**全部丢掉**，推理时换成用户手按的离散动作。这是全文最该停下来想三遍的设计。

**三大组件**
- **隐动作模型 LAM**（300M）：见上。输入是**像素**不是 token（消融证明像素更可控，见下）。patch 16，码本 8 码、embedding 32。
- **视频分词器**（200M）：VQ-VAE，把 $T$ 帧 $x_{1:T}\in\mathbb{R}^{T\times H\times W\times C}$ 压成离散 token $z_{1:T}$。用 ST-transformer 做编码器**和**解码器（论文叫 ST-ViViT），所以每个 $z_t$ 因为因果结构包含了 $x_{1:t}$ 全部历史信息——比只压空间的方法画质好。patch 4，码本 1024 码、embedding 32。
- **动力学模型**（最终 10.1B）：decoder-only 的 [[MASKVIT_ MASKED VISUAL PRE-TRAINING FOR|MaskGIT]] transformer。吃 $z_{1:t-1}$ 和 stopgrad 的隐动作 $\tilde a_{1:t-1}$，预测下一帧 token $\hat z_t$。训练时按 Bernoulli($0.5\!\sim\!1$) 随机 mask 输入 token，交叉熵损失。
  细节：隐动作和动力学都把动作做成**加性 embedding**（additive），不是常见的 concatenate 到帧上——作者发现加性更可控。

**ST-transformer：为什么这架构是关键** 普通 transformer 每个 token 看所有 token，视频 token 多达 $O(10^4)$，二次复杂度直接爆。ST-transformer（[[Tan_Temporal_Attention_Unit_Towards_Efficient_Spatiotemporal_Predictive_Learning_CVPR_2023_paper|时空注意力]]思路）把注意力拆开：空间层只在单帧内 $1\times H\times W$ 个 token 上算；时间层只在同一位置跨 $T$ 帧的 $T\times1\times1$ 上算，且加因果 mask。主导成本（空间注意力）随帧数**线性**增长而非二次。每个 ST block 只在时空两层之后放**一个** FFW（省下来的容量留给其它组件——作者说这显著提升效果）。三个组件全用这架构。

**训练流程** 两阶段、标准自回归视频生成管线：先单独训视频分词器；再**联合训** LAM（直接从像素）和动力学模型（在 token 上）。

**推理：怎么"玩"** 给一张起始图 $x_1$ → 分词器编码成 $z_1$ → 玩家选一个离散动作 $a_1\in[0,8)$ → 查那个 8 码码本得到 $\tilde a_1$ → 动力学模型预测 $z_2$ → 解码器还原成 $\hat x_2$ → 重复。自回归一帧帧滚。可以喂真实视频反推出的动作来重放原视频，也可以改动作生成全新轨迹。可条件于多张提示帧，论文为简化从单图起步。

**创新点清单**
- 第一个**纯视频**（无动作、无文本标签）训练出的、可**逐帧**操控的生成式交互环境（见 Table 1 与世界模型/视频模型对比）。
- **无监督隐动作**：用 VQ-VAE 瓶颈 + "解码器看不到未来"的约束，把可操控动作从像素逼出来；码本压到 8 保证人类可玩。
- **隐动作可迁移**：从互联网视频学的隐动作，能给完全没见过的环境（CoinRun RL）的专家视频打标，训出模仿策略。打通"无限互联网视频 → 训练通用智能体"的路。
- **ST-ViViT 分词器**：编解码器都用时空 transformer，时间感知的 token，画质更高且省内存。
- **动作做加性 embedding** 而非拼接，提升可控性。
- 系统性 **scaling 研究**：40M→2.7B 动力学模型，loss 随算力优雅下降；最终 10.1B。

**借用的经典研究（站在谁的肩膀上）**
- **VQ-VAE**（van den Oord 2017）：离散表示学习。分词器和隐动作量化都靠它。
- **MaskGIT**（Chang 2022）：掩码生成式 transformer，动力学模型的骨架。
- **ST-transformer**（Xu 2020）：原本做交通流预测的时空 transformer，被搬来当所有组件的底座。
- **ViT**（Dosovitskiy 2021）/ **Transformer**（Vaswani 2017）：基础。
- **C-ViViT / Phenaki**（Villegas 2023）：时间感知分词器的前身，被 Genie 用更省算力的 ST-ViViT 比下去。
- **VPT**（Baker 2022）：用逆动力学 + 人标动作给互联网视频打标——Genie 的对照组，区别是 Genie **完全不要**真实动作标签。
- **Playable Video Generation / PVG**（Menapace 2021）：隐动作控制视频的先驱，但只针对领域内静态样本，不能 prompt 生成全新环境。
- **世界模型**（Ha & Schmidhuber 2018；[[GAIA-1 world model_|GAIA-1]] 自动驾驶、UniSim 机器人）：但那些都要文本/动作标签，Genie 只要视频。
- 关联本地论文：[[RoboFlow4D A Lightweight Flow World Model Toward Real-Time Flow-Guided Robotic Manipulation|RoboFlow4D]]、[[The Surprising Effectiveness of Linear Models|线性模型]]、[[Rethinking the Objectives of Vector-Quantized Tokenizers for Image Synthesis|VQ 分词器目标]]。

**有趣的涌现现象**
- **视差 parallax**（Fig 12）：平台游戏常见的前景比背景动得快。Genie 没被教过，自己学会了，等于隐含理解了 3D 场景。
- **可形变物体**（Fig 11）：机器人数据上学会模拟薯片袋这类软物体的物理形变。
- **隐动作语义一致**（Fig 13/17）：不同起始帧下，同一个隐动作码始终对应同一语义（左/右/跳/不动、down/up/left）——证明它学的是"动作"而不是"这张图的特例"。脚注 2 说得好：搞清每个隐动作干嘛，就像在新手柄上摸清每个按钮。
- **强 OOD 泛化**（Fig 10/16）：训练全是游戏录像，却能玩进 Imagen2 文生图、手绘草图、真实照片——只在大规模数据上才学得到这种鲁棒性。

**实验与消融**
- **LAM 输入：像素 > token**（Table 2）。token 输入 FVD 略好，但可控性 $\Delta_t\text{PSNR}$ 明显更差——分词可能丢掉了动态信息，隐动作模型吃原始视频更好。
- **分词器架构：ST-ViViT > ViT > C-ViViT**（Table 3）。ST-ViViT FVD 81.4、内存 0.9GB，综合最好；C-ViViT 全空时注意力，内存 1.6GB 还过拟合、FVD 272.7 最差。
- **数据质量 > 数量**（Table 4）：精选 6.8M 视频（30k 小时）FVD 54.8，打败原始 55M 视频的 61.4。先手标 10k 视频训个 ResNet18 当质量分类器再筛。
- **隐动作可迁移到 RL**（Fig 15）：CoinRun 上用冻结 LAM 给专家视频打标训 BC，只需 **200 条**带真实动作的样本做隐→真映射，就追平能看到全部真实动作的 oracle。
- **机器人**：RT1 数据上训 2.5B 模型，FVD 82.7，学出一致的 down/up/left。

**关键数字速查**
| 项 | 值 |
| 原始数据 | 200k+ 小时互联网游戏视频 |
| 精选训练集 | Platformers 6.8M 段 16s 视频 ≈ 30k 小时，160×90，10FPS |
| 最终模型 | 动力学 10.1B + 分词器 + 动作 = 总 10.7B |
| 隐动作码本 | $|A|=8$ |
| 分词器码本 | 1024 码，patch 4，embedding 32 |
| 序列长度 | 16 帧 |
| 推理 | 每帧 25 步 MaskGIT，温度 2 |
| 可控性指标 | $\Delta_t\text{PSNR}=\text{PSNR}(x_t,\hat x_t)-\text{PSNR}(x_t,\hat x_t')$，越大越可控（真动作 vs 随机动作的差） |
| 视频质量指标 | FVD（Frechet Video Distance） |

**局限（作者自己承认的）**
- 只有 **16 帧记忆** → 长程一致性差，玩久了会忘。
- **约 1 FPS** → 交互太慢，离实时游戏远。
- 会**幻想出不真实的未来**，继承自回归 transformer 的通病。
- 未放出权重、数据、样例（出于安全考虑）；附录 F 给了单卡可复现的小版本（CoinRun）。

> [!question] 费曼拷问：真懂还是货物崇拜
> 我把这篇按我的法子盘一遍，标出哪些是真东西、哪些还没验证：
> - **真演示 > 论证**：Fig 13/17 那张"同一个动作码、不同起始图、动作语义一致"的图，是全文最硬的证据。它不是在论证"我们学到了动作"，它**让你看见**了。这是费曼会点头的地方。
> - **该说不知道的地方它说了**：脚注 2 老实承认"第一次交互时，没人知道每个隐动作会干嘛，得自己摸"。没装懂，好。
> - **一个没被它自己拷问的假设**：8 个动作够不够，是被"人类可玩性"这个**产品目标**逼出来的，不是被"环境真实自由度"逼出来的。平台游戏也许真就 8 个按钮，但换个世界（机器人连续控制）这个 8 就是个武断的竹子控制塔——附录 C.1 自己也说增加码数能提可控性但伤可玩性。这是个张力，不是定论。
> - **需要自己验证的说法**：FVD 这个指标到底多大程度对应"人觉得好玩/真实"？论文引 Unterthiner 2019 说它和人类评价对齐——但这是别人的实验，不是 Genie 自己测的。信，但标出来这是借来的信。
